# Redfish 自動化測試開發計畫

## 目標
- 建立以 `pytest` 為基礎的 Redfish 自動化測試框架，參考 OpenBMC 的 Redfish 支援行為。
- 支援 sensor 門檻值（threshold）驗證與邊界測試。
- 提供實時進度回報、每項測試標題與結果，並輸出專業 HTML/PDF 測試報告，包含原始 Redfish Response 與 Log 作為證據。

## 技術選型
- Python 3.9+
- 測試框架：`pytest`
- HTTP / Redfish client：`requests`（自訂 wrapper）或選用 `sushy` / `python-redfish`
- 報表：`pytest-html`（基礎）+ 自訂 HTML/PDF 補充（可用 `weasyprint` 或 `wkhtmltopdf`）
- 進度顯示：`rich` 或 `tqdm`
- 設定/門檻檔：`YAML`（`PyYAML`） + `jsonschema` 驗證

## 高階步驟
1. 確定需求與測試目標（目標 BMC IP/憑證、sensor 清單與門檻值）
2. 建立專案骨架與環境（virtualenv、`requirements.txt`、`tests/`）
3. 實作 Redfish client wrapper（認證、重試、Timeout、raw response 儲存與日誌）
4. 設計 `pytest` fixtures 與測試結構（target、credentials、thresholds 載入）
5. 實作 Sensor 門檻值測試模組（正常、邊界、異常情境）
6. 進度與狀態回報（CLI 實時進度、每項測試標題/結果）
7. 報表產生（概要、逐項結果、原始回應附檔、圖表）
8. 證據保存（raw JSON、pytest log、serial log、截圖）
9. CI/CD 範例（GitHub Actions）
10. 文件（`README.md`、使用說明、範例執行）
11. 示範執行並交付範例報告
